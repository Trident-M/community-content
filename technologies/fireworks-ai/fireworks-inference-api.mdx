# Fireworks Inference API

The Fireworks Inference API lets developers call open-weight language, vision, and embedding models through a single OpenAI-compatible endpoint at `https://api.fireworks.ai/inference/v1`. Switching between models is a one-line change: every model is addressed by an ID like `accounts/fireworks/models/deepseek-v3p1`, so the same client code can route requests to different models at runtime. Existing OpenAI SDK code works by pointing `base_url` at Fireworks and swapping in a Fireworks API key.

## Start building with the Fireworks Inference API

Serverless inference needs zero setup: pick a model ID from the [model library](https://fireworks.ai/models), send a request to `/chat/completions`, and pay per token. The `/models` endpoint lists which models are callable serverlessly at any moment, which is worth checking before building around a specific model, since some models in the catalog require a dedicated on-demand deployment instead. On-demand deployments are billed by GPU-second rather than per token and offer lower latency, no hard rate limits, and a broader model selection including custom model uploads.

Requests can be made with the native Fireworks Python SDK, the OpenAI Python or JavaScript SDKs, or plain HTTP. Streaming is supported with `stream=True`, and closing the connection stops generation so you are not billed for ungenerated tokens. See what the community has built with Fireworks at [lablab.ai](https://lablab.ai/tech).

### Key Use Cases

**Multi-model routing agents**
Because every model sits behind the same API and differs only by model ID, routing agents can send easy queries to a small, cheap model and escalate hard ones to a larger model without changing client code. Keeping model IDs in environment variables makes swapping tiers a config change rather than a code change.

**Token-scored hackathon submissions**
AMD-sponsored hackathons on lablab.ai route all answer-generating inference through the Fireworks API via a `FIREWORKS_BASE_URL` environment variable, with token usage scored directly. Members of the AMD AI Developer Program receive $50 in Fireworks credits. Explore [upcoming AI hackathons](https://lablab.ai/events) to find events built on Fireworks.

**Agent and tool-calling pipelines**
Function calling, JSON mode, and grammar-constrained generation let models return machine-parseable output directly, without extra parsing logic between the model and the rest of the pipeline.

**Drop-in OpenAI migration**
Teams with existing OpenAI SDK code can test open-weight models by changing only the base URL, API key, and model ID, keeping the rest of the integration untouched.
